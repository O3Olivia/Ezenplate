## 👩🏻‍💻 프로젝트 소개
이젠플레이트는 지역별로 맛집 리스트를 제공해주는 서비스입니다. <br/>
맛집 리스트, 검색 필터링, 리뷰, 평점 등 종합적인 맛집 경험을 제공합니다.<br/>
특히 지역 게시판을 통해 지역별로 사용자가 자유롭게 소통할 수 있는 커뮤니티 기능을 제공하여 기존 맛집 소개 서비스와 차별화를 두었습니다.

## 👾 주요 개발 이슈
### Issue 1 - 페이지네이션 기능

**🚨 Situation**
- 정보제공 처리가 중요한 프로젝트라 `SEO`를 고려했습니다.
- 그에 따라 주요 컨텐츠는 `SSR`을 선택했으며, 댓글 기능의 경우 `UI/UX`를 위해 페이지네이션 기능을 추가했습니다.


**💡 Solution**
- 댓글 수가 증가하게 되면, 이를 여러번 나눠 사용자에게 보여주는 것이 **화면의 복잡성을 줄이고 빠른 서비스 제공**할 수 있다 생각했습니다.<br/>
따라서 댓글 리스트는 페이지네이션 기능을 사용했습니다.<br/>
- 서버 부하를 줄이기 위해 한 번에 최대 **10개**씩 요청할 수 있도록 API를 구성했습니다.<br/>
- 이후, 페이지네이션으로 `더보기` 버튼을 생성해 **초기 로딩 시간을 줄이고, `UI/UX`상 유저 경험을 향상**시킬 수 있었습니다.

### Issue 2 - 반복되는 영역 모듈화
**🚨 Situation**
- 프로젝트 기술 스택은 **Spring**과 **Jsp**로 진행되었습니다. 
- 페이지마다 동일한 기능과 디자인의 `header`, `footer`, `navigation`이 사용되었습니다.
- 그러나 각 페이지의 레이아웃에 따라 해당 부분의 디자인과 기능이 지속적으로 변경되었습니다.
- 또한, 해당 기능을 구현하기 위해 각 페이지마다 동일한 **코드 중복**이 발생했습니다.

**💡 Solution**
- 디자인과 기능이 지속적으로 변경됨에 따라 `header`, `footer`, `navigation` 레이아웃을 **재사용**할 수 있도록 모듈화했습니다.
- 공통 부분을 `common`폴더를 생성해 관리했으며, 해당 부분이 필요한 경우 import하여 사용할 수 있도록 변경했습니다.
- 해당 코드를 재사용할 수 있게되면서 불필요한 **코드 중복**을 없애고, 디자인과 기능을 통일화시켰습니다.
